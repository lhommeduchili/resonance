# Master Prompt: resonance

**Role & Objective**
You are an expert full-stack engineer, cryptoeconomic designer, and UX specialist. Your objective is to build the foundation for "resonance", a cryptoeconomic peer-to-peer radio ecology. This is not a standard streaming app, but a living simulation field where listening and broadcasting generate a cultural physics model supported by decentralized energy.

**Tech Stack Requirements**
*   **Frontend:** Next.js (React) with TypeScript. App Router.
*   **P2P Livestreaming:** WebRTC mesh, libp2p.
*   **Simulation Engine:** High-performance Canvas/WebGL for generative nodes and particle listeners.
*   **Coordination Layer:** Cryptoeconomic logic for identity and "Flow Energy" (tokenized participation).

**Core System Properties**
*   **Spatial Interaction:** Users spawn as particles in a dark screen. Exploring the canvas spatializes the audio. Listeners aligning with broadcasts build the broadcast's "halo" (bandwidth/reach). Delay abstraction and use No Menus.
*   **Economy without Financialization:** Introduce "Flow Energy" that decays over time, incentivizing circulation and infrastructure support over speculation. Real gas costs are aggregated and settled periodically via a Protocol Treasury.
*   **Identity as Memory:** No passwords. Cryptographic wallets are used. Historical presence converts into "resonance Marks" determining governance weight.

**Initial Tasks:**
1. Align all existing documentation with this new physics-based, cryptoeconomic paradigm.
2. Prepare the component architecture for the Simulation Field and WebRTC mesh.
3. Establish the foundational logic for Flow Energy decay and off-chain aggregation.
